Playwright + POM + Portfolio

Гайд по 20 автотестам для QA Automation портфолио

Пошаговая шпаргалка по твоему проекту Padel Community: куда класть тесты, как использовать fixtures и POM, какие ожидания писать и как должен выглядеть код.

Проект: padel-community

Стек: Playwright, TypeScript, Page Object Model, test fixtures

Главный принцип: тест описывает бизнес-сценарий, POM хранит детали UI.

Базовые правила

Reset уже централизован в tests/fixtures/test.ts. Если тест использует page или любой POM fixture, reset вызывается автоматически. Если тест использует только request, добавь test.beforeEach(async ({ resetApp }) => await resetApp()). 
import { test, expect } from "./fixtures/test";
import { users, games, clubs, fullName } from "./data/test-data";

test.describe("Feature area", () => {
  test("[category] scenario name", async ({ loginPage, homePage }) => {
    await loginPage.loginAndExpectHome(users.sasha.email);
    await homePage.openGame(games.open);
  });
});
  • Не импортируй test из @playwright/test в e2e specs - используй ./fixtures/test.
  • Не используй waitForTimeout. Жди URL, alert, visible/hidden, enabled/disabled, текст.
  • Если локатор повторяется - добавь метод в POM.
  • Если тест меняет БД - reset нужен. Сейчас он уже встроен в page fixture.
  • Имена тестов оставляй бизнесовыми: что проверяем и какой риск закрываем.

1. [functional] user can update profile and changes persist after relogin

tests/auth.spec.ts

Зачем: Проверяет end-to-end поток изменения профиля и сохранение данных после новой авторизации.

Подсказки:

  • Используй ProfilePage, не пиши локаторы профиля прямо в тесте.
  • После submit жди success alert или обновлённое значение в profile card.
  • Persistence лучше проверять после logout/login, а не только сразу после save.

Полезный POM/helper:

// tests/pages/profile-page.ts
async expectProfileContains(...values: string[]) {
  for (const value of values) {
    await expect(this.card).toContainText(value);
  }
}

Пример теста:

test("[functional] user can update profile and changes persist after relogin", async ({
  loginPage,
  profilePage,
  page
}) => {
  await loginPage.loginAndExpectHome(users.sasha.email);

  await profilePage.goto();
  await profilePage.updateProfile({ city: "Tel Aviv", phone: "0501234567" });
  await expect(profilePage.success).toBeVisible();
  await profilePage.expectProfileContains("Tel Aviv", "0501234567");

  await page.getByTestId("avatar-menu-button").click();
  await page.getByTestId("logout-button").click();

  await loginPage.loginAndExpectHome(users.sasha.email);
  await profilePage.goto();
  await profilePage.expectProfileContains("Tel Aviv", "0501234567");
});

3. [functional] organizer can edit game club and price is recalculated

tests/organizer.spec.ts

Зачем: Проверяет редактирование клуба/города и автопересчёт цены в форме игры.

Подсказки:

  • Не кликай поля формы из теста - сделай метод updateGame в OrganizerPage.
  • Жди redirect ?saved=game после сохранения.
  • Проверяй и address, и price, потому что оба зависят от выбранного клуба.

Полезный POM/helper:

// tests/pages/organizer-page.ts
async updateGame(input: { city?: string; clubLabel?: string; courtNumber?: string }) {
  if (input.city) await this.testId("game-city").selectOption(input.city);
  if (input.clubLabel) await this.testId("game-club").selectOption({ label: input.clubLabel });
  if (input.courtNumber) await this.testId("game-courtNumber").fill(input.courtNumber);

  await Promise.all([
    this.page.waitForURL(/\/organizer\/games\/[^/]+\?saved=game$/),
    this.testId("game-save").click()
  ]);
  await expect(this.success).toHaveText("Game details saved.");
}

Пример теста:

test("[functional] organizer can edit game club and price is recalculated", async ({
  loginPage,
  organizerPage,
  page
}) => {
  await loginPage.loginAndExpectHome(users.organizer.email);
  await organizerPage.goto();
  await organizerPage.openGame(games.open);

  await organizerPage.updateGame({
    city: clubs.herzliya.city,
    clubLabel: clubs.herzliya.name
  });

  await expect(page.getByTestId("game-address")).toHaveValue(/Ramat Yam/i);
  await expect(page.getByTestId("game-price")).toHaveValue(/₪/);
});

12. [regression] completed tournament updates ranking after page refresh

tests/ranking.spec.ts

Зачем: Проверяет, что ranking не только обновился, но и переживает refresh.

Подсказки:

  • Можно переиспользовать поток создания турнира из admin tests.

  • Проверяй конкретного игрока и ожидаемые points.

test("[regression] completed tournament updates ranking after page refresh", async ({

19. [accessibility] critical pages have no blocking accessibility violations

Пример теста:

import AxeBuilder from "@axe-core/playwright";
import { test, expect } from "./fixtures/test";

for (const path of ["/", "/ranking", "/login", "/register"]) {

    await page.goto(path);

    const results = await new AxeBuilder({ page })
      .withTags(["wcag2a", "wcag2aa"])
      .analyze();

    expect(results.violations).toEqual([]);
  });
}

  • Тест не зависит от результата предыдущего теста.
  • Новые действия вынесены в POM, если они повторяются.
  • Тестовые данные взяты из tests/data/test-data.ts, а не разбросаны строками.
  • В API-only тестах reset вызван явно.
